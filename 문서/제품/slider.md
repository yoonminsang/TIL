- slider의 변화량이 항상 같을 필요는 없다. 1부터 10까지는 1씩 커지고 10부터 50까지 10씩 커질수도 있다.
- slider에서 이상 이하를 넣어야하는 경우가 있다.
  - 예를들면 대부분의 경우는 1에서 10안에 들어오는데 정말 예외적인 경우만 10 초과라고 가정해보자.

  - min, max 상태가 있다고 가정하자.
  - 1. max가 min이하로 갈 수 없게 막는 경우
  - 2. max가 min이하로 가면 min도 같이 작아지는 경우
  - 3. max가 min이하로 가면 max와 min이 뒤바뀌는 경우

# 🎚️ 슬라이더(Slider/Range) 컴포넌트 설계 가이드

웹 프론트엔드 개발 시 슬라이더 컴포넌트의 확장성과 사용자 경험(UX)을 고려한 핵심 설계 로직입니다.

---

## 1. 스케일 및 스텝 설계 (Non-linear Steps)
데이터의 밀도나 중요도에 따라 슬라이더의 변화량을 가변적으로 설정합니다.

* **비선형 스케일 구현**: 특정 구간에서 `step` 값을 다르게 적용합니다.
    * *예: 1~10 구간은 step 1, 10~50 구간은 step 10.*
* **구현 팁**: 
    * 내부 상태값은 0~100(%) 선형 데이터로 관리하고, 출력 시에만 변환 함수(Mapping Function)를 통해 실제 값을 계산하는 것이 로직 관리에 용이합니다.
    * 사용자가 변화를 인지할 수 있도록 주요 지점에 **Tick(눈금)**을 배치하는 것이 좋습니다.

## 2. 경계값 처리 (Boundary Values & Infinity)
정해진 범위를 넘어서는 예외적인 값을 상수로 처리하여 API 효율성을 높입니다.

* **상수 활용**: 최대값(Max)을 임계점으로 정의하고, 해당 지점에 도달 시 특수 처리를 수행합니다.
* **데이터 컨버팅 (API Mapping)**:
    * UI 상에서 `MAX`에 도달하면 `10+` 혹은 `제한 없음`으로 표시.
    * API 전송 시에는 `Infinity`, `null`, 혹은 `999,999`와 같은 약속된 값으로 변환하여 전송합니다.

## 3. 핸들 충돌 정책 (Collision Policies)
Range Slider에서 두 핸들(Min, Max)이 교차되거나 만날 때의 동작을 세 가지 유형으로 정의합니다.

### ① 차단형 (Prevent Collision)
* **동작**: Max가 Min 값 이하로 내려가지 못하도록 이동 범위를 제한합니다.
* **특징**: 가장 안정적이며 데이터 역전을 물리적으로 방지합니다.

### ② 밀기형 (Push)
* **동작**: Max를 왼쪽으로 밀어 Min과 겹치면, Min도 Max와 동일하게 같이 작아집니다.
* **특징**: 선택한 범위(Interval)를 유지하며 전체 구간을 이동시키고 싶을 때 유용합니다.

### ③ 교차형 (Swap)
* **동작**: Max가 Min보다 작아지는 순간, 두 핸들의 역할(Min ↔ Max)을 즉시 바꿉니다.
* **특징**: 조작이 자유롭고 사용자가 빠르게 범위를 재설정할 수 있습니다.

## 4. 프론트엔드 구현 체크리스트
- [ ] **Accessibility**: 스크린 리더가 `10+` 같은 커스텀 레이블을 인식할 수 있도록 `aria-valuetext`를 설정했는가?
- [ ] **State Management**: `onChange` 시 매번 연산을 수행하지 않도록 최적화(Debounce 등)가 필요한가?
- [ ] **Visual Feedback**: 핸들이 겹치거나 밀릴 때 사용자에게 시각적으로 명확한 피드백을 주는가?
