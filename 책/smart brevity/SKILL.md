# Smart Brevity

## 핵심 원칙

> 간결은 자신감이다. 장황은 두려움이다.

**독자가 먼저다.** 내가 말하고 싶은 것보다 상대가 알아야 할 것을 써라.

스마트 브레비티는 성격이 아니라 도구다 — 사람은 따뜻하게, 문서와 메시지는 간결하게.

## 4단계 프레임워크

### 1. 도발 (Headline)
짧고 강력한 한 문장. 독자의 시선을 사로잡아라.
- ❌ "CEO 외로움에 관한 연구 결과"
- ✅ "당신의 CEO는 점심 친구가 없다"

### 2. 리드 (Lead)
첫 문장 = 독자가 모르는 것, 알고 싶은 것, 알아야 할 것.
- 직설적이고 짧고 날카롭게
- 이 한 문장만 읽어도 핵심이 전달되어야 한다

### 3. 왜 중요한가 (Why It Matters)
맥락을 제공하라. 독자는 모르는 걸 질문하기 부끄럽다.
- 헤드라인 + 리드 + 왜 중요한가 = 곧바로 이해 가능해야 한다
- 유용한 경구: 큰 그림 / 앞으로 일어날 일 / 뒷이야기 / 행간 읽기

### 4. 깊이 알아보기 (Deep Dive)
더 알고 싶은 사람을 위한 것. 강요하지 마라.
- 불릿, 볼드, 경구를 활용하라
- 긴 문단은 피하라

## 작성 프로세스

1. **목록** — 반드시 전달할 요점을 적는다
2. **줄여라** — 1-2가지로 압축 (줄이기 어려우면 불릿으로)
3. **지워라** — 필요 없는 단어를 모두 삭제
4. **확인** — "이 한 문장만 보면, 전달하려 한 내용인가?"
5. **멈춰라** — 너무 많이 말하는 것이 최대의 낭비

## 상황별 적용

| 상황 | 적용 방법 |
|------|----------|
| 이메일 | 제목 = 도발, 첫 줄 = 왜 읽어야 하는지 |
| 회의 | 목적 1줄 + 어젠다 최대 3불릿을 사전 공유 |
| 프레젠테이션 | 슬라이드당 최소 단어 + 이미지 |
| 피드백 | 핵심을 직접 말하되, 상대와 맥락을 고려 |
| 소셜/뉴스레터 | 첫 줄에 모든 것을 걸어라 |

## 포맷 규칙

- **볼드**로 핵심 강조
- 긴 문단 → 불릿으로 분해
- 능동태 동사 사용
- 약한 단어·부사·수식어 제거
- 짧은 문장이 언제나 긴 문장보다 낫다

## 흔한 실수

| 실수 | 해결 |
|------|------|
| 결론보다 맥락을 먼저 씀 | 요점부터, 맥락은 뒤에 |
| 독자가 아닌 나를 위해 씀 | "상대가 알아야 할 것"으로 질문 전환 |
| 열심히 만든 것은 못 지움 | 필요 없으면 무조건 삭제 |
| 격식 때문에 핵심이 흐려짐 | 친구에게 말하듯 써라 |
